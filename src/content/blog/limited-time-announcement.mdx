This post has both a `post_hero.date` (in the past) and a `post_hero.unpublish_date` (in the future). It is inside its publish window, so it is live right now — it has a page and appears in the blog index, tag pages, the RSS feed, and "Recent Posts" lists.

A post is only included in the build while `date <= now < unpublish_date`. Because the unpublish date is still ahead, `scripts/generate-schedule.mjs` adds an `Unpublish Post` line for it to `_schedule.txt` in the build output. CloudCannon reads that file and schedules a build at the timestamp. Once that build runs, the unpublish date has passed, the publish gate in `src/utils/posts.ts` excludes the post, and it drops off the site entirely — no manual step required.

Leave the unpublish date empty to keep a post published indefinitely. Change it to the past and rebuild to see this post disappear immediately.
